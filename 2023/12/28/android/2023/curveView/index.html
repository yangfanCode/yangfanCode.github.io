Android实现LightRoom曲线绘制

杨帆2023-12-282023-12-30

小米图片编辑-曲线模块是此版本开发中较为复杂的功能点，主要涉及到曲线的函数算法、view的绘制、手势控制、控制点与曲线边界值处理等，本文对实现方案做系分拆解

1. 目标

针对用户的需求我们的产品应该达到以下几点：

  1. 曲线调整参数准确
  2. 滑动手势流畅、无卡顿、触摸灵敏可控
  3. 控制点拖拽操作流畅
    [图片]

2. 业务分析

曲线工具功能强大，特别是对于色彩这一块占着不可动摇的地位，其功能性被广大设计者尊称为调色之王。
而曲线原理本文不做详细阐述，简单总结，就是通过曲线的坐标改变，使一张直方图映射为另一张直方图，从而改变图像的效果。

[图片]

3. 实现方案

3.1 曲线算法

3.1.1 掐头去尾

满足曲线中性灰特性压缩暗部、扩展暗部、压缩亮部等，首个控制点之前和最后控制点之后都是一条纵坐标不变的直线

备注
Lightroom 目前只有有暗部压缩是水平直线，高光压缩仍可以自由调整高光的层次。为了使暗部更加平滑，高光部分增加灵活性。用户可以通过调整曲线高光的角度来改变高光区域的对比度和明暗程度，进一步优化图像的细节和色彩，比如：来处理同样处于高光带的天空和云彩的细节。

首尾坐标控制

###3.1.2 算中间
根据给定的点集（x, y）构造一条光滑的曲线，通过计算在数组 t 位的坐标处的插值来输出对应的数组 z 位纵坐标。

  1. 给定一组点（xj,yj），其中 j∈[0,n−1]，我们需要构造一个光滑的曲线 S(x) 在这些点上进行插值。这里采用三次样条函数进行插值，在每个小区间 [xj,xj+1] 内拟合出一个三次多项式 Sj(x)，使得曲线在每个插值点处的函数值、一阶导数值和二阶导数值均相等，这样可以保证整个曲线在每个插值点都具有良好的光滑性。我们可以将每个小区间内的三次多项式表示为：

Sj(x)=aj+bj(x−xj)+cj(x−xj)2+dj(x−xj)3

其中，j表示第j个小区间，aj、bj、cj、dj分别为第j个小区间内的四个系数。
2. 在每个小区间两个端点处，函数值、一阶导数值和二阶导数值均相等，即：

Sj(xj) = yj
Sj(xj+1) = yj+1
S’j(xj+1) = S’j+1(xj+1)
S’’j(xj+1) = S’’j+1(xj+1)

  1. 样条函数在每个小区间内的一、二阶导数分别为：

S’(x) = Sj’(x) = bj + 2cj(x - xj) + 3dj(x - xj)^2
S’’(x) = Sj’’(x) = 2cj + 6dj(x - xj)

方程组具有三对角矩阵结构，可以使用托马斯算法（也称为TDMA算法）进行求解。经过求解，我们可以得到每个小区间内的四个系数aj、bj、cj、dj，从而得到整条曲线S(x) 的表达式。

  1. 最后，对于任意的插值点 t，找到其所在的小区间[xj,xj+1]，然后使用该小区间的Sj(x)计算出对应的插值结果Sj(t)即可。

  1. 控制点小于2个，就是一条[0,255]的直线

3.2 绘制

考虑到四条曲线分别有不同的层级、不同的控制点与样式，使用4个画笔对象【All】【RED】【GREEN】【BLUE】分别对应4条曲线

在onDraw方法内绘制总体分为三步：

  1. 先绘制背景色
  2. 绘制网格背景
  3. 绘制曲线和控制点

3.2.1 绘制背景和网格

背景色直接填充即可
网格的计算思路为

  1. 确定网格线条数
  2. 通过 view 宽度计算网格线偏移量
  3. 使用canvas.drawLine方法绘制网格 注意：这条线的宽度是同时向两边进行扩展的，例如绘制一个圆时，将其宽度设置为 120 则会向外扩展 60 ，向内缩进 60，如下图
    [图片]
    因此如果绘制的内容比较靠近视图边缘，使用了比较粗的描边的情况下，一定要注意和边缘保持一定距离(边距>StrokeWidth/2) 以保证内容不会被剪裁掉。

[图片]

3.2.2 绘制曲线和控制点

曲线

  1. 通过上述样条曲线算法计算曲线纵坐标集合，横坐标为(0,255)，
  2. 通过view的宽度、高度计算(0,255)每个x、y 坐标所占像素大小
  3. 从起点开始向第二个点画一段直线，以此类推
    暂时无法在飞书文档外展示此内容

[图片]
控制点

  1. 根据控制点坐标集合确定坐标点在屏幕中位置
  2. 根据坐标绘制圆形控制点（选中状态需要保存记录）

[图片]

3.3 手势

3.3.1 控制点操作

判断手指触点是否点击在控制点上，决定高亮控制点还是插入控制点

[图片]

3.3.2 曲线重绘

  1. 获取当前点击的控制点边界值
  2. 更新控制点集合内的控制点坐标，刷新 UI

注意

  1. 曲线绘制的顺序问题，需要保证当前高亮的曲线层级始终在最上面。
  2. 控制点高亮的数据存储问题
  3. 曲线滑动边界值、控制点边界值的偏移量的算法控制

3.4 预设处理

曲线会给用户一些默认的模版预设，此处的思路为

  1. 首先要知道用户想要什么风格，才能确定想要控制直方图中的哪一部分区域，从而确定每条曲线控制点的坐标
  2. 控制点坐标确定，曲线的形态自然就生成了，即控制点确定曲线
